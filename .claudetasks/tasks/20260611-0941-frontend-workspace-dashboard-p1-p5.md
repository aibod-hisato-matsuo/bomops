**P1 基盤:**
- Vite + React 19 + TypeScript 5.9 を `frontend/web/` にスキャフォールド

- Vite dev proxy（/api → Django:8000）

**P2 Workspace読み取り:**
- 共通基盤: useList/useDetail/useGet フック、URL同期フィルタ、DataTable（ブランド準拠表デザイン）、Pagination、Badge、FilterBar、Tabs
- 5画面: 部品マスタ / 部品実物 / 製品モデル(+BOM詳細) / 顧客・拠点(タブ) / BSSセット(+詳細: 搭載部品・有効コンフィグ・イベント履歴)

**P3 Workspace書き込み:**
- バックエンド: page_size パラメータ対応、ProtectedError→409 例外ハンドラ
- useCreate/useUpdate/useDelete + DRFフィールドエラーのフォームマッピング + トースト通知
- 全リソースのドロワー型 作成/編集フォーム（react-hook-form + zod）
- 搭載/取外し（部品ステータス連動）、SiteConfig エディタ（secret は dirty フィールドのみ送信でマスク値上書き事故を防止）、イベント追記（追記型UI）、BOM行追加/削除

**P4 検索・使用履歴:**
- バックエンド: `GET /part-units/{id}/history/`（購入/搭載/取外し/保守を時系列統合）、lookup に part_unit_id 追加
- 横断検索画面（シリアル逆引きカード + 部品/セット/拠点/顧客の横断検索）
- 使用履歴タイムライン画面（色分けマーカー付き縦タイムライン）

**P5 Dashboard:**
- バックエンド: `GET /dashboard/summary/`（セット/部品/拠点/顧客の集計を1レスポンス）
- KPIカード4種 + Recharts棒グラフ4種 + 直近イベントフィード、60秒自動更新
- recharts はルート遅延ロードでチャンク分離（メインバンドル肥大化を回避）
