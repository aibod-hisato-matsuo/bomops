## 📝 作業サマリー

used_in の ProductBOM 導出化により概念の流れが「製品モデル（レシピ正本）→ 部品マスタ（参照される部品）」になったことを受け、ユーザー提案どおりサイドメニューの順序を製品モデル先頭に変更した。一貫性のためホームの関係マップも製品モデル起点に反転した。

**サイドメニュー（AppLayout）:**
- マスタ（静的）: 製品モデル → 部品マスタ の順に変更
- 運用（動的）: BSSセット → 部品実物 → 顧客・拠点 の順に変更（左メニュー全体が「製品系（製品モデル→BSSセット）／部品系（部品マスタ→部品実物）」の縦系列で揃う。BSSセット=製品モデルの実体、部品実物=部品マスタの実体という対応）

**ホーム関係マップ（WorkspaceHome）:**
- マスタ帯: `製品モデル ─BOMで部品を参照→ 部品マスタ`（矢印の向き・ラベルを「モデルが部品を参照」に修正）
- 層間コネクタ: 左列 ↓型番を選択 / 右列 ↓実物化 1:N（シリアル採番）
- 運用帯: `BSSセット ─実物を搭載→ 部品実物`、顧客・拠点は BSSセット の下（↑設置 1:N）へ移動
- 結果、「左上の製品モデル → 右へ部品 → 下へ実体」という設計→部品表→実物調達の業務フロー順に読めるマップになり、メニューとマップの順序が完全一致
