## 📝 作業サマリー

baiten-gcp プロジェクトへの実プロビジョニングを実行し、BOMOps 検証環境を公開・E2E検証まで完了した。

**公開URL:**
- SPA: https://baiten-gcp.web.app （Firebase Hosting）
- API: https://bomops-api-cught7d4hq-an.a.run.app （Cloud Run, asia-northeast1）
- `/api/**` は Hosting rewrite で Cloud Run へ転送（同一オリジン・CORS不要）

**プロビジョニング内容:**
- API有効化（Run/SQL/Artifact Registry/Cloud Build/Secret Manager）
- Artifact Registry リポジトリ `bomops`
- Cloud SQL `bomops-pg`（PostgreSQL 16, db-f1-micro, enterprise edition, 東京）
- Secret Manager 3種（Django SECRET_KEY / BOMOPS_ENCRYPTION_KEY / DBパスワード）
- 実行SAへ cloudsql.client / secretAccessor 付与
- Cloud Build でイメージビルド（53秒）→ マイグレーションJob実行（13テーブル適用）→ Cloud Run デプロイ（ゼロスケール, 512Mi, max 3）
- スーパーユーザー作成Job実行（パスワードは別途共有・初回ログイン後に変更指示済み）
- baiten-gcp を Firebase プロジェクトとして登録し Hosting デプロイ

**発生した問題と解決:**
1. 請求未リンク → ユーザーが billing 権限を自己付与後、CLI からリンク成功
2. プロジェクト権限不足（billing.projectManager/run.admin のみ・人間のowner不在）→ ユーザーが owner 付与、IAM伝播 約20秒待ちで解決
3. **Cloud SQL `db-f1-micro` が 400 エラー** — 新しい gcloud は Enterprise Plus エディションが既定で共有コアTier不可 → `setup.sh` に `--edition enterprise` を追加して解決
4. baiten-gcp が Firebase 未登録 → `firebase projects:addfirebase` で登録

**E2E検証結果:** SPA 200 / health（直接・rewrite経由とも）200 / JWT ログイン（rewrite経由）成功。

**ランニングコスト:** Cloud SQL db-f1-micro 約1,500円/月が主。Cloud Run はゼロスケール。

**残課題（本番化前）:** IAP（Workspace SSO）前段追加、DBティア引き上げ、CI/CD設定、admin初期パスワード変更。
