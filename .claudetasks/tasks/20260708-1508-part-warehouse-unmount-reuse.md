部品を故障で取り外す運用と、外した部品の保管先（倉庫）、別セットへの再利用までを一連で扱えるようにした。本番デプロイ済み（Cloud Run rev bomops-api-00012-5qg、マイグレーション0008適用）。

**設計判断（倉庫＝CustomerSite の再利用）:**
- データ調査で AIBOD大名/大熊オフィスが 顧客=AIBOD の `lifecycle_status=拠点(BASE)` 拠点として既存と判明 → 新モデルを作らず **倉庫 = BASE拠点の CustomerSite** とする（設置先と保管先を同一の場所概念で統一・データ移行ゼロ）
- `PartUnit.storage_site`（CustomerSite への nullable FK・migration 0008）＝未搭載時の物理保管先

**取外しフローの格上げ（UnmountComponentDrawer 新規）:**
- セット詳細の「取外し」を確認ダイアログ → ドロワー化。取外し後ステータス（在庫/故障/廃棄）＋保管先倉庫（BASE拠点から選択）＋保守イベント記録（既定ON・故障→FAILURE/他→REPLACEMENT）を一括実行。故障品が誤って在庫に戻る穴を解消

**現在の場所の可視化:**
- 部品実物の列を「使用中セット」→「現在の場所」に拡張: 搭載中はセットリンク／未搭載は 🏭 倉庫名／どちらも無ければ未登録
- PartUnit シリアライザに `storage_site`（書込可）/`storage_site_name`。実物フォームにも保管先倉庫セレクト追加

**別セットへの再利用（取外し→搭載）:**
- 既存の「取外し（在庫化）→ 別セット詳細で部品を搭載（在庫一覧から選択）」で成立。使用履歴に A→B の遍歴が残り、現在の場所がセットBに更新
- 修正: 搭載時に stale な storage_site を null クリア（搭載中の部品が倉庫も持つ不整合を防止）
