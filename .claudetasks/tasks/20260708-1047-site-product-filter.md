## 📝 作業サマリー

顧客タブと同様に、拠点タブにも製品フィルタボタンと「設置製品」列を追加した。顧客と違い、拠点は物理的な場所なので **設置実績からの導出のみ**（手動登録は持たない）とした。

**設計判断:**
- 拠点の製品関係は `拠点 → 設置された製品セット → 製品モデル → ファミリ` から100%導出できる。設置ゼロの拠点は本当にまだ何も無いため、顧客に足した「手動M2M」は拠点には不要と判断
- 「準備中で実機がない拠点」を製品に紐づけたいケースは、拠点ではなく顧客側の手動タグで表現するのが正しい（場所に付くのは実機の事実、商談・契約は取引相手の関係）

**バックエンド:**
- `CustomerSiteSerializer.products`（設置実績由来の製品ファミリ名リスト・読み取り専用）を追加。N+1回避に `prefetch_related("sets__product_model__family")`
- `CustomerSiteFilter.product_family`（`sets__product_model__family__name`・distinct）追加
- 新API `/customer-sites/product-summary/`（製品ファミリ×拠点数集計・既存 CustomerProductSummarySerializer を再利用）
- テスト3件新規（導出・集計・フィルタ）— **57/57 パス**

**フロントエンド:**
- CustomersPage: 製品CascadeRow を顧客/拠点の両タブ対応に一般化（各タブの集計・総数に切替）。拠点タブにも「製品:」ピルボタン行を表示
- 拠点一覧に「設置製品」バッジ列（全て実績=sky。顧客のような実績/手動の色分けは不要）

**動作確認（ユーザー実操作込み）:** 松本工業/豊前工場（設置0台=製品null）に対し、ユーザーが製品セット `RSCV-0068-0001`（RVB-STD-8G / RISC-V Board）を豊前工場へ設置作成 → 拠点の設置製品が `RISC-V Board` に自動反映、拠点フィルタに `RISC-V Board (1拠点)`、松本工業も RISC-V 顧客として実績伝播することを確認。lint/フロント23/23・build クリーン。

**補足で助言した運用ポイント:** セット status を設置済(INSTALLED)にすると稼働カウントに乗る／実機セット作成後は顧客の手動タグは冗長／構成部品を搭載すると Set Twin が完全になる。
